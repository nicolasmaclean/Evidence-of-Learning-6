Yet again I found a tutorial in my YouTube recommendations. This is a common occurrence throughout most of my learning, but I also lean by having to search more specific topics, while creating projects or experimenting. The project shows the progress that I had made from watching the first 3 videos of an 8 part tutorial series by Chris Courses. The first 2 videos taught how to create canvas in HTML and draw on it JavaScript. It only included how to draw basic shapes like rectangles, circles, and lines. However, in the third video I learned how to create basic animations. A method to create animations on a canvas is to create animation loop. The loop will run at a specific frame rate and updates based on other code created by the user. So to move a circle from the left side of the screen to the right, the x position of the circle must increase by a given amount each frame. I also learned my first bit of physics simulation in video. Mouse Event Listeners

This video taught me how to interact with the canvas using mouse movement through event listeners. Events Listeners are created in JavaScript to to "listen" for specific events. So a click event listener will wait for the user to click on a specific element and run a function when they do. I first learned how to use a click event listener with the rock, paper, scissors, lizard, spock project, but during this one I learned how to use mouse movement. To take in the mouse input I added a mousemove event listener. It updates a variable with the x and y position of the mouse anytime it moves. I can then take this data and do some fun stuff with it later on with other code. In this project I started out with the same code, almost, as the previous project to have circles bouncing around the screen randomly. However, the main difference is how each circle updates in each frame. If the circle is not close to the mouse, it shrinks to minimum size and if it close to the mouse, it grows.
I set the minimum size of each circle to be 0, so they would not appear to the user showing a blank canvas. As the mouse moves around, though, circles nearby grow in size and become visible. Above I show another version of the original code with the minimum size set to a visible value to show what is going on. Interacting with HTML Canvas Video

Collision Detection and Physics

I learned the basics of collision detection from sixth and seventh videos of the tutorial series by Chris Courses. It is a very basic version of collision detection that can only detect if 2 circles are overlapping. It can't detect where on the circles the collision occurs and only works if both objects are circles. To calculate if the 2 circles are colliding, I used a little bit of trigonometry to calculate the distances of the circles' centers. If this distance is less than the sum of their radii, the circles are intersecting.
During the second part of the video mentioned, he shows how to simulate 2 circles or balls colliding and bouncing off each other in a realistic manner. He used the same method of collision detection from the previous paragraphs. In the video, he uses some pre-made code to calculate the velocities of each ball after collision that I don't quite understand. He gave a brief explanation of the logic, but did not really explain the code. I could not really understand how he calculated it, so I spent some time to do my own research and learn how to do it from other mediums. I ended up making the above code using equations to calculate the physics portion of the code that I found on a forum. The code is still very buggy and does not work as well I as I wanted it to, but I already spent way too much time on it so I have left it as is. The collision detection is not perfect, so the circles get caught in each other causing some very strange looking bugs. I think the equations might also be off a little bit off too, but I'm not sure if it is just the collision detection causing all the bugs.

Circular Motion

This project was made while following the last video of the Chris Courses' series about circular motion. Throughout the animation I have made thus far, I had split all coordinates and velocities into 2 components: an x and y. So for each object, there was an x coordinate, y coordinate, x velocity, and y velocity. I also added a mouse event listener to allow the circles to revolve around the cursor instead of a set point. A circular motion can also be decomposed into x and y components. Each one oscillates from positive to negative values, so they are best represented with sine and cosine functions. I decided to model the x motion with cosine and y motion with cosine. There are other combinations, but this is what I came upon first.
While I was creating this, my calculus class was going over the concept of polar functions, which are very similar to this topic, So I created a polar graphing calculator. This program allows you to change the polar equation inside the code and has the particle travel along the graph of it. Circular Motion Video

Spring Animation

This project followed a video created by Val Head about creating a spring motion most commonly seen in things that chase the mouse cursor. It uses simplified physics, so it is not realistic but it looks close enough. The program relies on a target position, the mouse, and the current position. It calculates a force based on how far it is from the target and moves it with this force towards the target. But the force will increase the ball's speed as it moves towards the target position causing it to overshoot it. The ball will continue going past the target, but it will slow down until it reverses directions and head back towards the target position. This process repeats till the ball stops at the target. I added range sliders at the bottom of the screen to allow the user to play around with different variables and affect the ball's motion. The drag variable plays the role of friction, decreasing the velocity allowing it to stop. The spring constant variable is strength of the spring, so the higher the value the harder to pull the spring. The trail was added for fun and the higher the number the faster the trail disappears. The project was supposed to just be a quick, fun physics simulation, but with experimentation and the range sliders it take a lot longer. This project helped me put another physics simulation in my toolbox and helped me learn how to use range sliders and integrate it JavaScript. Spring animation Video
